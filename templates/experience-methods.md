# 경험 소재화 방법론 (experience-methods)

경험을 서류·면접용 소재로 바꾸는 공통 절차의 단일 정의. 4년/60건+ 첨삭에서 검증된 방법론이며, ETHOS.md가 철학(왜)이라면 이 문서는 실행 절차(어떻게)다. 소비 스킬은 해당 Phase에서 "§N을 적용하라" 형태로 섹션 단위로 인용한다.

---

## §1. 경험 전환 6단계

**적용 시점**: 소재 발굴·초안 작성 (경험 1건을 카드 1장으로 구조화할 때)

| 단계 | 질문 | 산출 |
|---|---|---|
| 1. 이름 | 이 경험을 한 줄로 부르면? | 경험 제목 |
| 2. 문제 | 무엇이 문제였나? (누가·언제·얼마나) | 문제 정의 |
| 3. 역할 | 그중 내 몫은 정확히 어디까지였나? | 역할 범위 |
| 4. 바꾼 행동 | 내가 무엇을 다르게 했나? | 구체 행동 |
| 5. 검증 가능한 변화 | 무엇이 달라졌고, 어떻게 확인했나? | before→after |
| 6. 직무 연결 | 지원 직무의 어떤 일과 닿아 있나? | 직무 태그 |

6단계 중 하나라도 비어 있으면 그 경험은 아직 소재가 아니다 — 비어 있는 단계를 질문으로 채운다.

## §2. 문제·역할·행동·결과 4분리

**적용 시점**: 초안 작성·첨삭 (한 문장에 여러 층위가 섞였을 때)

문제/역할/행동/결과를 각각 별도 문장으로 분리한다. 팀 성과와 본인 기여를 한 문장에 섞지 않는다.

**피해야 할 표현 5종** (감지 즉시 4분리로 재작성):

| 피할 표현 | 문제점 |
|---|---|
| "다양한 경험을 통해" | 경험이 특정되지 않음 |
| "책임감을 가지고" | 행동이 없음 |
| "소통 능력을 발휘해" | 무엇을 조율했는지 없음 |
| "많은 것을 배웠습니다" | 결과가 없음 |
| "팀에 기여했습니다" | 본인 몫이 없음 |

## §3. 수치 폴백 5기준 + 대체 4종

**적용 시점**: 수치화 코칭 (성과 숫자가 없다고 할 때 — 날조 금지, guardrails §1)

숫자가 없어도 아래 5기준 순서로 근거를 찾는다:

1. **전후 변화** — 정성이라도 before→after ("수기 정리 → 양식화")
2. **역할 범위 분리** — 전체 중 내 몫 ("API 12개 중 8개 담당")
3. **정성 근거** — 사수 피드백, 내가 만든 양식이 계속 쓰임
4. **작은 검증 가능 숫자** — "3주간 문의 12건 유형 정리"가 "수백 건 처리"보다 낫다
5. **면접 설명 가능성** — 이 숫자를 면접에서 1분간 설명할 수 있는가? 설명 불가한 수치는 쓰지 않는다

**수치 대체 4종**: 범위(담당 API 개수) · 빈도(주 1회 배포) · 전후 비교 · 담당 규모(고객사 15곳)

## §4. 추상어→질문 전환표

**적용 시점**: 첨삭 (추상어를 발견하면 삭제하지 말고 질문으로 전환해 구체 경험을 캐낸다)

| 추상어 | 전환 질문 |
|---|---|
| 책임감 | 끝까지 맡아서 처리한 일이 무엇인가? |
| 소통 | 누구와 누구 사이의 이해 차이를 줄였나? |
| 꼼꼼함 | 어떤 오류·누락을 얼마나 줄였나? |
| 문제해결력 | 문제를 어떻게 정의했나? |
| 성장 | 그 전과 후에 무엇이 달라졌나? |

## §5. 약한 문장 5유형 + 보강

**적용 시점**: 첨삭 (문장 단위 진단)

| 약한 문장 유형 | 보강 방향 |
|---|---|
| "~을 좋아합니다" | 좋아해서 실제로 한 행동 사례로 교체 |
| "열심히 했습니다" | 역할을 구체화 (무엇을·어디까지) |
| "성장했습니다" | 전후 비교로 교체 |
| "소통했습니다" | 조율 유형을 특정 (전달·정리·설득·조정 중 무엇) |
| "책임감 있게" | 맡은 범위 + 문제 발생 시 처리 기준으로 교체 |

## §6. 어조 전환 3공식

**적용 시점**: 최종 다듬기 (ETHOS.md의 3공식과 동일 — 여기서는 적용 순서만 정의)

| 공식 | Before | After |
|---|---|---|
| 희망→실행 | "~하고 싶습니다" | "~하고 있습니다" / "~해 왔습니다" |
| 감정→행동 | "흥미를 느꼈습니다" | 구체적 프로젝트와 성과 서술 |
| 과정→결과 | "밤새 공부했습니다" | "응답시간 350ms → 15ms 개선을 달성했습니다" |

적용 순서: 문장별로 ①어느 공식에 해당하는지 판정 → ②After 형으로 재작성 → ③재작성에 필요한 사실이 없으면 §4의 질문으로 사용자에게 확인 (추정 금지).
